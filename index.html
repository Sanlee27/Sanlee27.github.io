Model, View, Controller를 각각 나누어 구현하는
Model2 방식을 이용해 만든 개인 프로젝트로, JAVA Calendar API를 사용해 만든 가계부입니다.

사용자와 가계내역, 해시태그와 접속자 수 총 4개의 테이블이 사용되었습니다.
로그인은 Session을 통해 구현했으며, 최초 로그인 시 Session을 생성하고,
해당 날짜의 접속자(counter) 테이블에 추가되어 일일/누적 접속자 수에 합산되도록 하였습니다.
가계 내역 작성 시, 수입과 지출 내역을 구분 선택하고
메모에 해시태그(#)를 붙혀 작성하면, 메인 상단에 해당 키워드가 표시되도록 했습니다.

달력
- Java의 Calendar API를 사용해 년/월/일을 구해 달력을 구현했습니다.

CRUD
- View에서 EL을 사용하여 코드를 구현하였습니다.
- c:set을 사용, 변수명에 값을 할당하여 출력되도록 하는 등, JSTL core의 태그를 사용해 구현했습니다.

현재/일일/누적 접속자
- Listner를 이용해 현재 이용중인 고객의 수를 확인할 수 있습니다.
- Session이 소멸될 경우 console에 표시, 접속자 수를 줄도록 하였습니다.

성장점

  • Model1 구조에서 Form과 Action으로 나누어 처리하던 것을
    Controller의 doGet과 doPost로 처리할 수 있다는 것을 알게되었습니다.
  • View코드와 Model, Controller 등의 로직 처리 코드가 분리되어 있어,
    유지보수가 용이하고, Model1구조에 비해 jsp가 복잡하지 않는 등의
    Model2 구조의 장점에 대해 알게 되었습니다.
